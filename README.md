# MONEY — личный финансовый помощник

Telegram-бот + веб мини-апп для учёта личных финансов одного пользователя.
Принимает чеки (фото с QR → ФНС), текст, голос и банковские выписки, сам всё
категоризирует нейросетью и показывает дашборд: траты, капитал, цели, прогнозы.

- Бот: **@money_veta_bot** · Мини-апп: **https://money.vetaone.site**
- Боевой сервер: `166.88.159.91` (контейнер `money_app` за общим Caddy)

> README — актуальное состояние сервиса. [SPEC.md](SPEC.md) — исходная спецификация/видение.

## Возможности

**Ввод (бот):**
- 🧾 Фото чека с QR → распознаём QR → API ФНС `/scan` → состав по товарам.
- 📷 Фото без QR (ценник/квитанция) → vision-нейросеть определяет трату.
- ✍️ Текст: «такси 300», «зарплата 135000», «дал другу 5000 в долг».
- 🎙️ Голос → расшифровка (Gemini) → как текст.
- 📄 CSV-выписка (Райф/1С/Сбер/Google Sheets — **авто-детект разделителя** `;`/`,`/Tab) → импорт со склейкой, дедупом и категоризацией. **Heuristic «перевод → transfer» работает только для расходов** — приходы с merchant'ом «Перевод денежных средств» остаются как `income`, направление не теряется. Импорт **не корректирует balance счёта** (исторические выписки давали ложное +170k к актуальному балансу) — после импорта обновляешь balance вручную в Капитале.
- 🆕 **Ручной ввод операции через `POST /api/tx`** теперь **автоматически корректирует balance** счёта: income +amount к источнику, expense −amount, transfer −amount у source и +amount у counterparty. Снимок капитала today force-обновляется. Удобно для зарплат через крипту/наличку, которые в выписку не попадают.

**Обработка:**
- Категоризация по позициям: правила (`CategoryRule`) → LLM (Gemini) → порог уверенности; **учится** на правках.
- **Двусторонняя склейка чек↔выписка**: чек находит операцию выписки по сумме ±72ч и обогащает её составом (и наоборот при импорте выписки); дубли исключены (по `fn+fd+fp` и по сумме±время). Под распознанной операцией в боте — кнопка «📂 Открыть в приложении» (deep-link к этой операции).
- Возвраты (чек `operationType=2`) → минус-расход. Долги (занял/дал) → влияют на капитал, не на траты.

**Мини-апп (6 вкладок):**
- **Главная** — safe-to-spend; четыре цифры месяца (потрачено/доход/отложено/капитал) со стрелками Δ к прошлому месяцу в одной строке с заголовком (компактно); «N трат ждут разбора»; прогноз месяца; график трат по месяцам; категории с Δ и **мини-спарклайнами трендов за 6 мес** (клик → операции категории); последние операции + «Все операции»; **«поговорить с ассистентом»** (значок чата в шапке → Gemini-чат по своим данным) + **шестерёнка → Настройки**.
- **Операции** — полный список с фильтрами (период/тип/категория/поиск по продавцу/сумма от-до/«на разбор»), группировкой по дням с дневным итогом, итогом по выборке, пагинацией; **клик по продавцу** → его операции; **bulk edit** (выделить N → массовая правка категории); тап → правка категорий + заметка; ручное добавление операции с **smart-compose** (печатаешь «Маг» → подсказка «Магнит · Продукты · ~700 ₽», тап и форма заполнена) включая **перевод между счетами**; метка возвратов; **реклассификация СБП-переводов** через блок «Что это было?» с 6 вариантами (Покупка/расход · Доход · Перевод между счетами · Я дал в долг · Вернули мой долг · Я погасил долг) — для transfer'ов с СБП по номеру: один тап превращает в expense с категорией, в self-transfer со счётом-получателем, в новый Debt или закрытие существующего; правило сохраняется → следующие переводы тому же merchant'у авто-классифицируются при импорте.
- **Аналитика** — **финансовое здоровье** (норма сбережений + **скользящие 3/6/12 мес**, подушка, USD/RUB, персональные рекомендации) + **календарь трат за 6 мес** (heatmap по дням, **прижат к правому краю**: свежие недели всегда видны с нормальным отступом, старые недели обрезаются слева — тап по квадрату → операции этого дня) + **«Что я заметил»** (аномалии трат, spending velocity, топ-3 крупных, аудит подписок, pace к цели, ближайшие списания); **бюджет месяца** (лимит по категории: свой или авто-прогноз; план-факт с прогресс-барами; свод доход−расходы−цели = профицит); переключатель периода (месяц/квартал/год); доход/расход/накоплено + норма сбережений; доходы-vs-расходы по месяцам (тап по столбцу → операции месяца); сравнение с прошлым периодом по категориям (Δ% + **год-к-году** когда история накопится); структура расходов (сегментированная полоса + 2-колоночные карточки с мини-иконкой); топ-продавцы; подписки с **countdown до след. платежа**.
- **Доходы** — источники дохода (зарплата, фриланс, пособия) с иконкой, **правкой имени/суммы/валюты/периода/владельца/срока**, планом-фактом за месяц, разбивкой «откуда пришло» (сегментированная полоса), владельцем (я/жена) и нуджами «пора продлить»; план дохода = сумма активных источников. У операции дохода — своя категория (Зарплата/Премия/Кешбэк…) и привязка к источнику вручную (с обучением — впредь привязывается автоматически).
- **Капитал** — net worth (мультивалюта по курсу ЦБ) с Δ; **компактный график курса USD/RUB за год** (текущий курс ЦБ, Δ% за год, мин/макс, hover crosshair); линия динамики капитала; **карточка «Прирост»** с переключателем периодов (День/Неделя/Месяц/Год) и компактной шапкой (число под заголовком, чипы и дата справа); раскладка delta = Сбережения + Курс $ + Прочее, согласованная для выбранного периода; **🔥 Финансовая независимость (FIRE) под РФ в одном блоке с прогнозом**: 3 одинаковых серых плитки сценариев SWR (Осторожный 2%/50× · Базовый РФ 3%/33× · Оптимистичный 4%/25×) с числом лет до FI (28px) и правильным склонением («38 лет», «44 года», «21 год»), цель и SWR/множитель мелким серым, бейдж прогресса в углу; компактная строка «Расходы X/год · Откладываю Y/мес · Реал. Z%»; Runway и Coast FI рядом; **мини-карточка «Доходность портфеля»** (фактическая средневзвешенная ставка vs план + сколько ₽/год недополучаешь + доля «работающего» капитала); **график прогноза капитала** горизонтальной чертой ниже — горизонт автоматически `max(20, лет_до_пенсии + 5)` (для 38-летнего мужчины = до пенсии 2058 + 5 лет = 32 года), пунктиры FI-целей и точки пересечения; **две линии**: сплошная — план (по заявленной номинальной доходности), пунктирная — факт (по средневзвешенной ставке портфеля); hover-tooltip показывает обе цифры с дельтой; **распределение по валюте** + дрейф от целевой аллокации (например 70/30 RUB/USD); **подушка безопасности** (мес расходов); **цель капитала** + прогноз достижения + «отложено в этом месяце»; счета с **бренд-цветами банков** (Тинькофф/Сбер/Райф/Альфа/ВТБ/Газпром/Озон/Яндекс/Почта/МТС), CRUD, баланс инлайн с **поддержкой копеек** (`step=0.01`); **долги** (несколько контрагентов, частичные погашения, прогресс «отдано/осталось», остаток в капитале); **вклады/инвестиции 3.0** — модель «положил со счёта-источника на инструмент с доходностью»: создание вклада списывает principal с balance счёта-источника, пополнения через DepositTopup тоже списывают из выбранного источника, net worth не меняется (это перекладывание из «лежит» в «работает»). Валюта вклада подтягивается из источника (рублёвый вклад из Сбер-карты RUB, USDT-стейкинг из криптокошелька). `_grow_compound`(сложный %) vs `_grow_simple`(линейный) считается по-настоящему от реальных DepositTopup или fallback monthly_topup; кнопка «+ Пополнить» с выбором источника и проверкой что хватит баланса.
- **Цели** — прогресс/прогноз достижения, **привязка цели к счёту** (сумма авто из баланса), создание целей; **карточка «В месяц на регулярные»** (приведение year→÷12, week→×4.33); **регулярные платежи** с countdown до след. платежа (мес: только число, год: день+месяц; «Сегодня/Завтра/N дн./N мес K дн.» в цвете чёрный→оранжевый→красный); auto-advance даты при просрочке + бот-нудж «Сегодня/Завтра платёж: …» в 10:30; авто-детект кандидатов из выписки.

**Настройки** (шестерёнка на Главной) — **iOS-style каркас** с навигацией по 12 секциям (главная = список из `.setrow` с иконкой/названием/1-строчным сабтайтлом/шевроном; каждая секция = саб-страница с `‹` back-кнопкой и компактными подсказками `.sethint` 11px):
- **👤 Профиль** — имя, дата рождения, пол (вынесено из FIRE-модала, теперь одной кнопкой Сохранить).
- **👥 Семья** — переименование «Я»/«Жена» + добавление новых членов (везде в чипах счетов/доходов/вкладов).
- **💰 Доходы** — список источников зарплаты с план-фактом, owner-чипом, периодом; тап → редактор источника.
- **🔁 Регулярные расходы** — подписки/ЖКХ/аренда с суммой/мес (приведение: year÷12, week×4.33); переход в редактор Целей.
- **🏦 Счета и активы** — балансы карт/наличных/крипты/вкладов; тап → форма счёта (live-preview сверху с иконкой типа и балансом; чипы типа Карта/Наличные/Крипта/Внешний; чипы валюты как иконки ₽/$/₮; iOS-style секции).
- **🎯 Финансовые цели** — объединяет цель капитала + все параметры FIRE (инфляция, доходность, лет до пенсии, аллокация) + размер подушки (мес расходов) одной кнопкой Сохранить.
- **📊 Бюджеты** — лимиты с маркерами «прогноз»/«перебор»; переход в Аналитику.
- **🏷️ Категории** — менеджер CRUD: имя, родитель, цвет, иконка из набора 32. **Цвета теперь генерируются по golden ratio** (137.508° между корневыми → соседние индексы максимально различимы) + **HSL-семьи** (дочерние получают вариацию цвета родителя ±14° hue, ±6% light): «вся машина в красных тонах», «вся еда в зелёных» — визуально читается как одна зона на графиках. 24-цветный fallback палитры на случай отсутствия Category.color.
- **🔔 Уведомления** — алерт «крупная трата» + подсказка про дайджесты и нуджи.
- **🎨 Тема** — Авто/Светлая/Тёмная с галочкой на активной (синхронизация между устройствами через TG CloudStorage).
- **📥 Данные** — поиск по чекам ФНС, экспорт CSV (готовится), бэкап БД (готовится).
- **ℹ️ О приложении** — ссылки на бот и веб.

**UX-плюшки:** **свайп между вкладками** (горизонтальный: влево → следующая, вправо → предыдущая; не срабатывает на скроллируемых зонах/графиках/при открытой шторке); глобальный **FAB «+»** на всех вкладках (автоскрывается при скролле вниз); **hover-тултипы для десктоп-Telegram** на всех графиках (heatmap, курс USD/RUB, линия капитала, cashflow, bar-чарт, hbar-сегменты, топ-продавцы) с vertical crosshair + **click-to-drill** (тап по дню/месяцу → операции с фильтром); **skeleton-loaders** только при первой загрузке вкладки; плавная анимация смены вкладок (fade/directional slide); **TG BackButton** автосинхронизируется со шторками; haptic-feedback в ключевых местах; **AI-ассистент** на Gemini по своим данным с историей вопросов (localStorage); **smart-compose** автоподсказки магазинов в форме новой операции; **поддержка fullscreen Mini App** (запуск с иконки рабочего стола) — Safe Area API отступает контент под телеграм-кнопки.

**Производительность — событийный кэш:** данные не риалтайм, поэтому GET-ответы кэшируются на сессию (с дедупом параллельных одинаковых запросов). Переключение между уже открытыми вкладками — мгновенное, без сети и без перестройки DOM (скролл сохраняется). Кэш сбрасывается **только по реальным событиям**: любая мутация (POST/DELETE) чистит его сразу; при возврате фокуса в апп дёргается дешёвый `/api/version` (отпечаток БД) и кэш сбрасывается, лишь если данные менялись извне (ввод через бота). Без слепого TTL.

**Расчёты:** **единая база расхода** `expected_monthly_expense` — среднее за N последних ПОЛНЫХ календарных месяцев по всем тратам (используется и в «можешь откладывать», и в прогнозе капитала, и на дашборде — раньше были 3 рассинхронизированные формулы); safe-to-spend = доход (сумма источников) − уже потрачено − **ещё предстоящие в этом месяце** регулярные (по next_date, без двойного вычета уже прошедших) − взносы в цели; capacity «можешь откладывать» = доход − expected_monthly_expense; net worth в рублях; персональная норма сбережений.

**Уведомления (по расписанию, МSK):** алерты в момент траты (крупная ≥ лимита, перебор бюджета по категории); нудж продления регулярных + «сегодня/завтра платёж X» + **нуджи об инактивности** «давно не грузил выписку (≥14 дн.)», «чеки ФНС не подтягиваются (≥14 дн.)», «балансы не обновлялись (≥21 дн.)», «курс USD устарел (≥7 дн.)», «учёт затих ≥5 дн.» — раз в неделю на kind, чтобы не спамить (10:30); дайджест ежедневно 21:00 / еженедельно вс 20:00 / месячно 1-го 10:00. Команды `/report`, `/week`, `/month`. Бэкап БД 03:30, **снимок капитала ровно в 00:01** (стабильная точка начала дня — не переписывается операциями/правками в течение дня), автообновление токена ФНС 04:00.

## Архитектура

```
Telegram ──webhook──▶ FastAPI (один процесс) ──▶ SQLite (WAL)
   │                     ├─ aiogram (бот)
   │                     ├─ REST API (/api/*) для мини-аппа
   │                     ├─ APScheduler (дайджесты, бэкап, снимок капитала)
   │                     └─ services: qr, fns, llm, categorize, ingest, …
   ▼
Caddy (общий на сервере) ──TLS──▶ money.vetaone.site → money_app:8000
```

- **Один процесс** Python: FastAPI отдаёт API + статику мини-аппа и принимает Telegram-webhook (без polling). На слабом VPS экономно.
- **БД — SQLite (WAL)** через SQLAlchemy. Один пользователь → отдельный сервер БД не нужен, бэкап = один файл; при росте меняется на Postgres сменой `DATABASE_URL`.
- **Frontend** — один файл `frontend/index.html` (ванильный HTML/CSS/JS + Telegram WebApp SDK, иконки Tabler инлайн). Отдаётся самим FastAPI (StaticFiles). Без сборки — надёжно на слабом сервере. Сессионный кэш GET-ответов в памяти + событийная инвалидация (см. «Производительность»).
- **Один адаптивный layout, не две версии.** Компоненты (карточки/графики) одинаковы везде, меняется только раскладка через CSS media (≥900px): нижний таб-бар → боковой sidebar; блоки в одну колонку → masonry-сетка из 2-3 колонок (автогруппировка плоского рендера в секции `.secwrap>.block` через MutationObserver — `gridify`, без правки загрузчиков); bottom-sheet → центрированная модалка. Мобайл/Telegram (<900px) не затронут. Новый блок пишется плоско (h2+card) и раскладывается сам.
- **Деплой** — Docker (контейнер `money_app`) в существующей сети Caddy `my_server_caddy_net`; **свой nginx/порты 80-443 не поднимаем** — маршрут даёт общий Caddy.

## Модель данных (SQLite)

`accounts` (счета: card/cash/deposit/crypto/external, валюта, владелец me/wife/m1/…, баланс; `interest_rate`/`interest_compound`/`interest_note` — dead-code после отката концепции «ставка на счёте») ·
`categories` (expense/income/transfer, иерархия parent_id; `color` назначается по golden ratio для корней + HSL-вариация для детей) ·
`transactions` (сумма+валюта+`base_amount_rub`, тип, категория, продавец, источник, `dedup_key`, counterparty_account_id для трансферов) ·
`transaction_items` (позиции чека, своя категория) ·
`receipts` (фискальные данные: fn/fd/fp, JSON ответа ФНС) ·
`category_rules` (выученные правила) · `recurring` (регулярные доходы/расходы = источники дохода, period, owner me/wife, next_date с auto-advance) ·
`goals` · `budgets` (лимиты по категориям) · `debts` (paid для частичного погашения) · `deposits` (`source_account_id` — откуда были взяты деньги; `currency` — валюта инструмента; `kind` = deposit/staking/other для иконки) · `deposit_topups` (история фактических пополнений вкладов: deposit_id, date, amount, `source_account_id`) · `net_worth_snapshots` (зафиксирован в 00:01, неизменен в течение дня) · `fx_rates` · `external_reports` (для сверки жены) · `settings` (key-value: `alert_big`, `networth_target`, FIRE-параметры, `emergency_target_months`, `family_members`, и др.).

## ФНС (lkdr.nalog.ru)

- Логин по SMS защищён Yandex SmartCaptcha и режется по не-RU IP (`blocked.ip`) → делается **вручную в браузере** на российском IP, токены импортируются в `data/tokens.json` (`token`, `refreshToken`, `sourceDeviceId`).
- `/scan` и refresh работают с любого IP (в т.ч. US-сервер). Refresh-токен долгоживущий → ручной логин редко.
- **Авто-обновление access-токена** по refresh при любой токен-ошибке (401/403, «срок действия истёк») + ежедневный прогрев (04:00). Если refresh умер — бот сам зовёт на повторный вход.
- ⚠️ Суммы в ответе `/scan` приходят **в рублях** (float), не в копейках.
- Если ФНС начнёт блокировать IP сервера — задать `FNS_PROXY` (RU-прокси), применится только к запросам ФНС.

## LLM

- **Gemini** (прямые ключи Google AI, `GEMINI_KEYS` через запятую) — голос, vision, парсинг текста, категоризация. Ротация ключей на 429/5xx. Модель `GEMINI_MODEL` (сейчас `gemini-2.5-flash`).
- **OpenRouter** (`OPENROUTER_PAID_KEY`, `PAID_MODEL`) — для тяжёлой аналитики.
- Все вызовы — на сервере; ключи наружу не уходят.

## Безопасность

- Аутентификация — два пути, оба пускают **только `OWNER_TG_ID`**: (1) Mini App — Telegram `initData` (HMAC токеном бота); (2) браузер на money.vetaone.site — **Telegram Login Widget** (подпись = SHA256(bot_token)) → httponly-cookie с подписанным сессионным токеном (HMAC + срок 30 дней). Всё в `app/security.py`; чужим — 401/403. Для Login Widget домен привязан к боту в @BotFather (`/setdomain`).
- Webhook — секрет `WEBHOOK_SECRET` (заголовок Telegram).
- Секреты в `.env` (gitignored). `tokens.json`, БД, фото чеков, референсы — не в гите.
- Бэкап БД — зашифрован (Fernet, ключ из `BACKUP_PASSPHRASE`), уходит в личку владельца.
