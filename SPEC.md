# MONEY — личный финансовый помощник. Спецификация v1

Telegram-бот (ввод) + веб мини-апп (дашборд) + аналитический слой. Личное использование (один пользователь), но схема многопользовательская на будущее.

---

## 1. Источники данных и единый приём («не важно как заношу»)

Один вход в бота → роутер по типу:

```
ФОТО → ищем QR
       ├─ QR есть, ФНС      → наш парсер + ФНС /scan → товары (item-level)
       ├─ QR не читается    → vision-LLM (Gemini): сумма, продавец, что куплено
       └─ QR нет (записка/квитанция/любое фото) → vision-LLM
ТЕКСТ → LLM парсит «потратил 300 наличкой на кофе» → транзакция
ГОЛОС → Gemini принимает аудио напрямую → текст → как ТЕКСТ
ВЫПИСКА (CSV/Excel из Райфа) → строки → склейка с чеками → категоризация
        ↓
   Транзакция → склейка/дедуп → категоризация (правила→LLM→спросить) → БД
```

**Склейка/дедуп (ключевой механизм «чек уточняет транзакцию»):**
- чек ↔ строка выписки: матч по `сумма + время ±N мин + продавец` → одна операция (выписка = скелет, чек = товары);
- чек ↔ чек: по фискальному ключу `fn+fd+fp` (повторная загрузка не дублируется);
- результат: одна транзакция, обогащённая позициями; часть позиций может уйти в другую категорию (пример: −1000 «Продукты», но средство для стёкол → «Хозтовары»).

---

## 2. Категоризация (цель — ноль кликов)

3 слоя:
1. **Память правил** — ключ `ИНН продавца + нормализованное имя товара` → категория. Совпало с высокой частотой → ставим молча.
2. **LLM-классификатор** (Gemini, батч на чек) — категория + уверенность + краткое обоснование.
3. **Порог уверенности** — высокая → молча; низкая → пометка «на разбор» / инлайн-кнопки в чате (топ-2-3 + «другое»).
4. **Обучение** — каждая правка обновляет память правил; после N повторов система предлагает автоправило («Магнит + *ПИВО* → Пивасик?»).

Дерево категорий — в `categories.json` (expense / income / transfer).

---

## 3. Модель данных (PostgreSQL)

- **users** — id, tg_id, name (пока один).
- **accounts** — id, name, type (`card|cash|deposit|crypto|external`), currency (`RUB|USD|USDT|...`), owner (`me|wife`), is_external (счёт жены), balance, archived.
- **categories** — id, name, parent_id, type (`expense|income|transfer`), icon, archived.
- **transactions** — id, account_id, datetime, amount, currency, **base_amount_rub**, fx_rate, type (`expense|income|transfer|debt`), category_id, merchant, counterparty_account_id (для переводов), source (`receipt|statement|text|voice|photo`), note, status (`confirmed|needs_review`), dedup_key, created_at.
- **transaction_items** — id, transaction_id, name, name_normalized, qty, price, sum, category_id. *(item-level из чеков)*
- **receipts** — id, transaction_id, fn, fd, fp, t, s, n, raw_qr, fns_json, kkt_owner, inn, retail_place.
- **category_rules** — id, match_inn, match_item_pattern, category_id, confidence, hits, auto.
- **recurring** — id, name, category_id, account_id, amount, currency, period (`monthly|weekly`), day, **start_date, end_date (nullable)**, type (`income|expense`), reminder, active. *(регулярка со сроком — под субсидии жены, подписки)*
- **deposits** — id, account_id, bank, principal, rate, term_start, term_end, capitalization, monthly_topup, owner.
- **goals** — id, name, target_amount, currency, target_date, current_amount, monthly_plan, status.
- **debts** — id, counterparty, direction (`i_owe|owed_to_me`), amount, currency, date, due_date, status (`open|closed`). *(сейчас пусто, схема на будущее)*
- **net_worth_snapshots** — id, date, total_rub, breakdown_json.
- **fx_rates** — date, currency, rate_rub. *(курсы USD/крипты для сведения к рублю)*
- **external_reports** — id, date, account_id (жена), reported_balance, reported_income, reported_spend, note. *(для сверки счёта жены)*
- **settings** — expected_monthly_income, base_currency (RUB), safe_to_spend_cfg, digest_cfg, backup_cfg, fns_tokens.

## 4. Расчётные модули

- **Net worth** — сумма всех счетов (карты + наличные + вклады + крипта − долги), мультивалюта → RUB по `fx_rates`. Снимки в `net_worth_snapshots` → график «богатею ли я».
- **Safe-to-spend** — `X = ожидаемый_доход_мес − уже_потрачено − предстоящие_обязательные (recurring) − плановый_взнос_в_цели`. Показываем `X` и `≈ X/оставшиеся_дни в день`, динамический пересчёт (потратил больше → дневной лимит падает).
- **Регулярные + прогноз** — детект повторов; прогноз на месяц по категориям = трейлинг-медиана за 3 мес → «ожидаемые расходы ≈ Z», кормит safe-to-spend.
- **Персональная инфляция** — индекс твоей корзины во времени + тренды цен по товарам (только из item-level). Страница «Инсайты» + проактивный пуш.
- **Цели** — план/факт + прогноз достижения («при темпе — к 14 ноября») + нудж. **Goal discovery**: из нормы сбережений система предлагает цели; дефолтная первая — финансовая подушка (3–6 мес расходов).
- **Сверка счёта жены** — ожидаемый остаток (его переводы + её приходы − её траты/вклады) vs заявленный → подсветка утечки. Фундамент в v1, глубокая логика — фаза 3.
- **Возвраты/долги** — возврат (чек n=2) = отрицательный расход; долги = движение, влияет на net worth, не на траты/доход.

## 5. LLM-слой

- **Бесплатный Gemini (ротация 8 ключей OpenRouter, failover на 429)** — голос (аудио нативно), фото без QR (vision), парсинг текста, категоризация. Объём ≈ бесплатно.
- **Платный OpenRouter (сильная модель)** — глубокая аналитика, инсайты, разбор аномалий, месячные отчёты, goal discovery.

---

## 6. Telegram-бот

Приём фото/текста/голоса/документа (выписка); инлайн-кнопки категорий при разборе; дайджесты (ежедневный короткий / еженедельный большой / месячный); алерты (перебор по категории, крупная трата, «N трат ждут разбора»); нудж по целям; команда бэкапа.

## 7. Mini-app (дашборд, React + Telegram WebApp SDK)

- **Главная**: net worth, **Safe-to-spend** (крупно, с дневным лимитом), расходы месяца vs прогноз, прогресс целей.
- **Транзакции**: список, фильтры, правка категории, ручная склейка.
- **Категории/аналитика**: разбивка, тренды, drill-down по категории и по товарам.
- **Инсайты**: персональная инфляция, регулярные, аномалии.
- **Цели**: план/факт, прогноз, goal discovery.
- **Счета / Net worth**: балансы, история капитала.
- **Семья**: сверка счёта жены.
- **Настройки**: доход, валюта, дайджесты, бэкап, токены ФНС.

---

## 8. Инфраструктура

- Хостинг: VPS США (Чикаго/Атланта) — бот, мини-апп, БД, LLM.
- **ФНС**: логин по SMS юзер делает вручную в браузере на RU IP (его телефон), токен импортируем; refresh долгоживущий. `/scan` и refresh — с US VPS; если ФНС заблокирует US IP — заворачиваем **только запросы к ФНС** через дешёвый RU-прокси.
- **Бэкап**: дамп БД (gzip, зашифрован паролем) раз в сутки в приватный Telegram-канал; храним последние ~14.
- Безопасность: токены/ключи в секретах; данные жены и финданные — приватный канал бэкапа.

## 9. Roadmap MVP

**Фаза 0 — инфра.** VPS, Postgres, скелет бота (aiogram), тест ФНС `/scan`+refresh с US VPS (+RU-прокси при блоке), импорт `categories.json`, заведение счетов и стартового net worth (по интервью).

**Фаза 1 — ядро (MVP).** Фото чека → QR → ФНС → товары → категоризация (правила+Gemini) → БД; ручной ввод текст/голос; инлайн-кнопки разбора; базовый дашборд (траты по категориям, список транзакций). → *Уже можно вести учёт.*

**Фаза 2 — полнота и контроль.** Импорт выписки + склейка/дедуп; регулярные + прогноз; **safe-to-spend**; цели + goal discovery; график net worth.

**Фаза 3 — аналитика.** Персональная инфляция; дайджесты + алерты; сверка счёта жены; бэкап в Telegram; инсайты (платный LLM).

**Фаза 4 — доводка.** Зрелость автоправил; мультивалютные уточнения; фото-без-QR через vision; полировка UI.
